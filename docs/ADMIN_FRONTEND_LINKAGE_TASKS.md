# 后台联动与流程优化任务清单

本文档用于收口当前“后台管理流程、前台联动、录入体验优化”的专项任务，作为接下来产品、设计、开发联动的执行清单。

当前明确决策：

- 前台改为“匿名可见”。
- 后台录入流程必须从“分散多页面跳转维护”改为“围绕一次商品建档或一次报价发布的工作流”。
- 后续实现优先围绕“系列中心、批次中心、低跳转、可批量、可 AI 辅助”推进。

状态约定：

- `[x]` 已完成
- `[ ]` 待开发
- `[-]` 暂不进入当前批次

## Priority 1：前台改为匿名可见

目标：统一文档、权限和页面行为，明确前台为公开浏览入口，但后台写操作仍严格受 RLS 和角色控制。

- `[ ]` 同步更新文档口径：将 `docs/ARCHITECTURE.md`、`docs/PRD.md`、`docs/README.md` 中“前台不是匿名公开站”的描述改为“前台匿名可见，后台登录管理”。
- `[ ]` 复核并确认 `supabase/migrations/005_frontend_anonymous_read_access.sql` 作为正式策略，而不是临时策略。
- `[ ]` 梳理匿名可见范围：产品、分类、品牌、公司、资料、报价摘要、动态可见；后台管理页、上传、写操作不可见。
- `[ ]` 前台增加匿名访问视角验收：未登录状态下可直接打开首页、产品列表、详情页、资料中心、报价中心、公司库、动态页。
- `[ ]` 后台和 Supabase 权限回归：匿名仅可读公开数据；`user` 仅读；`admin` 可写。

验收标准：

- 未登录用户可直接浏览前台核心页面。
- 匿名访问不会看到后台入口、编辑按钮和写操作能力。
- RLS 与 Storage policy 口径和文档完全一致。

## Priority 2：Excel 商品快速入库工作台

目标：新增一个“一站式 Excel 商品快速入库工作台”，让管理员可以在一个页面内完成 Excel 上传、字段解析、商品草稿生成、规格抽取、报价抽取、人工确认和批量入库。第一阶段优先适配真实业务中的“报价总表 + 多个型号详情 Sheet”结构，例如 KDS 厨房系列报价单。

当前后台已有基础能力：

- 产品管理页已支持产品型号、系列编码、系列名称、产品名称、公司、分类、品牌、产品类型、尺寸、芯片、内存、存储、系统、分辨率、POE、状态、标签、简介、配置摘要、规格 JSON。
- 文件资料页已支持上传文件，并关联产品、公司、报价批次、文件类型、资料类型、主资料、分类、来源表、标签。
- 报价管理页已支持报价批次、商品、公司、币种、发布时间、生效日期、阶梯价、报价附件、标准配置、固件备注、状态、备注。
- 后台看板已有“待补齐清单”，可识别缺图片、缺规格资料、缺报价的型号。

本阶段不推翻现有产品、文件、报价管理页，而是在其上新增一个业务录入入口。

### 2.1 工作台入口

- `[x]` 新增 `/admin/product-info/workbench` 商品快速入库工作台。
- `[x]` 后台菜单增加“快速入库工作台”入口。
- `[x]` 工作台以一个页面为主，不拆成多个独立路由。
- `[x]` 页面内部使用步骤条和卡片分区完成流程。
- `[ ]` 后台看板增加快捷入口：`Excel 导入商品`、`手动新建商品`、`补齐资料`、`发布报价`。
- `[ ]` 保留现有产品管理、文件资料、报价管理页面，作为高级维护入口。

页面结构：上传 Excel → 解析结果预览 → 商品草稿确认 → 报价确认 → 入库检查 → 确认入库。

### 2.2 支持的真实 Excel 结构

第一阶段优先支持“报价总表 + 型号详情 Sheet”的 Excel。

示例结构：

```text
报价
KDS1589T
KDS2189T
KDS2489T
KDS1585T
KDS2185T
KDS2485T
```

解析规则：

- `[x]` `报价` Sheet 作为报价总表。
- `[x]` 其他以型号命名的 Sheet 作为产品规格详情。
- `[x]` 如果 Sheet 名称与型号一致，直接作为 `model`。
- `[x]` 如果详情 Sheet 内存在 `Model No`，优先使用 `Model No` 作为型号。
- `[x]` 报价总表里的“机型”字段与详情 Sheet 的 `Model No` 做匹配。
- `[x]` 若报价总表有型号，但没有对应详情 Sheet，也允许生成商品草稿，但标记为“缺少详情 Sheet”。
- `[x]` 若详情 Sheet 有型号，但报价总表没有对应报价，也允许生成商品草稿，但标记为“缺少报价阶梯”。

### 2.3 报价总表解析

报价总表字段映射：

| Excel 字段 | 系统字段 | 说明 |
|---|---|---|
| 尺寸 | `size_inch` / `spec_json.panel_size` | 解析 15.6、21.5、24 等尺寸 |
| 机型 | `model` | 商品型号 |
| 产品图片 | 后续 `document.product_image` | 第一阶段先不提取图片 |
| 标准配置 | `summary_config_text` / `standard_config_text` | 商品配置摘要和报价标准配置 |
| 10台单价（USD） | `quote_tiers` | `minQuantity=10`，币种 USD |
| 100台单价（USD） | `quote_tiers` | `minQuantity=100`，币种 USD |
| 500台单价（USD） | `quote_tiers` | `minQuantity=500`，币种 USD |
| 1000台单价（USD） | `quote_tiers` | `minQuantity=1000`，币种 USD |
| 可选配件摇臂支架 | `remarks` | 第一阶段先进入报价备注 |
| 可选配件立柱支架 | `remarks` | 第一阶段先进入报价备注 |
| 备注/选配 | `remarks` | 报价备注、材料差价、固件备注 |

报价批次字段：

- `[x]` 批次标题：来自报价 Sheet 顶部标题；解析失败则使用 Excel 文件名。
- `[x]` 公司：从表头公司信息尝试识别；第一阶段由管理员在页面批量选择确认。
- `[x]` 币种：从默认设置选择，默认 USD。
- `[ ]` 发布时间：由管理员选择。
- `[-]` 原始 Excel 文件作为报价附件保存后续实现。

生成结果：

- `[x]` 每个型号生成一条商品草稿。
- `[x]` 每个型号生成一组报价草稿。
- `[x]` 每个价格列生成一条阶梯价。
- `[x]` 报价状态默认 `draft`，管理员可以在导入前切换为 `active`。

### 2.4 型号详情 Sheet 解析

每个型号详情 Sheet 负责补充商品规格。

基础字段映射：

| Excel 字段 | 系统字段 |
|---|---|
| Model No | `model` |
| Product type | `product_type` |
| Product Description | `name` / `description` |
| CPU | `chipset` / `spec_json.cpu` |
| RAM | `ram_gb` |
| Internal memory | `storage_gb` |
| Operation system | `os_name` / `os_version` |
| Panel Size | `size_inch` |
| Resolution | `resolution_width` / `resolution_height` |
| Luminance | `brightness_nits` |
| POE | `poe_supported` / `poe_standard` |
| 其他 key-value 字段 | `spec_json` |

解析策略：

- `[x]` 对固定英文标签做 key-value 解析。
- `[x]` 常用字段提取到产品主字段。
- `[x]` 其余字段尽量保留到 `spec_json`，避免资料丢失。
- `[x]` 在 `spec_json` 中记录 `excel_file_name`、`quote_sheet_name`、`source_sheet_name`。
- `[-]` 商品详情 Sheet 作为独立“规格书资料”记录后续实现。

### 2.5 商品草稿确认表格

Excel 解析后，不直接入库，先进入当前页面的“商品草稿确认表格”。

表格列：状态、型号、产品名称、系列、分类、公司、尺寸、芯片、RAM / 存储、分辨率、报价阶梯、错误提示 / 警告提示。

状态：`可入库`、`需确认`、`错误`、`已入库`、`已跳过`。

交互能力：

- `[x]` 支持表格内直接编辑字段。
- `[x]` 支持批量应用默认分类、品牌、公司、系列名称。
- `[x]` 支持只导入选中行。
- `[x]` 支持跳过错误行，不阻塞正确行入库。
- `[x]` 支持重复型号检测并提示。
- `[ ]` 支持重复型号处理：跳过、更新已有商品、作为新商品创建。
- `[ ]` 支持匹配不到的分类、品牌、公司在当前页面快速创建。

### 2.6 分类、品牌、公司匹配规则

第一阶段策略：

- `[x]` 品牌由管理员在工作台统一选择默认品牌。
- `[x]` 公司由管理员在工作台统一选择默认公司，也可逐行选择。

- `[ ]` 匹配不到时支持当前页面快速创建分类、品牌、公司。

### 2.8 报价确认与入库

报价解析后不另开页面，而是在当前商品草稿表格中显示报价阶梯，入库时同步创建报价。

已实现能力：

- `[x]` 支持解析 10 / 100 / 500 / 1000 等阶梯价。
- `[x]` 支持批量设置公司、报价币种、报价状态、生效日期。
- `[x]` 支持空价格自动忽略，不生成该阶梯价。
- `[x]` 支持导入后默认保存为草稿。
- `[x]` 支持确认后批量创建报价和报价阶梯价。

第一阶段处理选配项：可选配件、备注、固件包先写入 `remarks` 或 `firmware_note`，不强制进入 `quote_options`。后续在 Priority 3 的报价批次详情页中，再补完整选配项维护能力。

### 2.9 入库确认

入库动作：

- `[x]` 调用现有 `createProduct` 创建商品。
- `[x]` 创建商品后调用现有 `createQuote` 创建报价和阶梯价。
- `[x]` 已入库行标记为“已入库”。
- `[x]` 入库完成后刷新现有产品列表缓存。
- `[-]` 暂不做更新已有商品。
- `[-]` 暂不做原始 Excel 附件上传。
- `[-]` 暂不做图片提取。

### 2.10 一页完成的交互流程

```text
进入商品快速入库工作台
  ↓
上传 Excel
  ↓
系统识别 Sheet 结构
  ↓
解析报价总表
  ↓
解析型号详情 Sheet
  ↓
生成商品草稿表格
  ↓
生成报价阶梯信息
  ↓

  ↓
人工批量修正
  ↓
入库检查
  ↓
确认入库
  ↓
显示入库结果
```

- `[ ]` 规划 `/admin/quotes/:batchId` 报价批次详情页，统一维护批次头信息、报价行、阶梯价、选配项和附件。

- `[ ]` 明确“系列”和“变体”的前后台展示规则：前台详情主对象是系列，变体作为切换器；后台录入主对象也是系列。

建议实施方案：

验收标准：

- 后台产品和报价的主对象与数据模型保持一致。
- 前台详情和后台详情围绕同一业务对象，不再各自维护一套理解。

## Priority 4：统一查询层和动态治理

目标：减少前后台两套口径继续漂移，并让动态只展示对前台真正有意义的信息。

### 4.1 查询层统一

- `[ ]` 梳理 `apps/web-front/src/api/product-info.ts` 与 `apps/web-ele/src/api/product-info/*` 的重复逻辑。
- `[ ]` 设计统一的聚合查询层或共享查询模块，统一产品、详情、报价、资料、动态的口径。

建议方案：

1. 后台 API 负责写操作和管理态列表。
2. 前台 API 负责公开态聚合查询，但底层字段映射和状态规则抽到共享层。
3. 系列详情、报价批次详情、动态卡片都走统一聚合模型。

### 4.2 动态治理

- `[ ]` 重写动态生成规则，只保留“新品发布 / 有效报价更新 / 可见资料更新 / 管理员通知”四类。

- `[ ]` 把“发布报价批次”作为报价动态的主要来源。
- `[ ]` 增加动态可见性规则：停用系列、停用变体、不可见资料、无效报价不进入前台动态流。
- `[ ]` 后台增加“是否生成前台动态”的确认或预览能力。

验收标准：

- 前台动态不再出现重复、噪音、误导信息。
- 动态内容能真实反映“产品发布、资料上新、报价更新、平台通知”。
