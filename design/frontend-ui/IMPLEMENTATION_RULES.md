# 前台实现与验收规则

本文档用于约束前台页面的实现方式、移动端适配方式和验收标准，避免开发过程中出现“桌面稿能看、移动端口径不清、验收标准不一致”的问题。

## 适用范围

当前规则优先适用于首批前台核心页面：

- 首页
- 产品列表页
- 分类导航页
- 品牌导航页
- 商品详情页

以下内容当前不进入首批开发范围：

- 首页热门产品
- 首页最新动态
- 独立动态页 / 动态流
- 收藏、历史、对比、工作台
- 前台上传资料
- 申请报价 / 联系公司

## 开发规则

### 1. 页面功能必须保留桌面端主能力

移动端不是删功能版，而是重排版。首批移动端必须保留桌面端主链路能力：

- 首页保留统一搜索、分类入口、资料中心入口、报价中心入口、公司库入口
- 产品列表页保留搜索、筛选、资料入口、报价摘要、关联公司、更新时间
- 分类 / 品牌导航页保留目录式浏览和聚合入口
- 商品详情页保留规格参数、相关资料、报价记录、关联公司

### 2. 前台实现优先使用项目现有能力

前台页面开发优先复用项目现有技术栈和组件能力：

- 项目入口优先基于 `apps/web-front`
- 组件库优先使用 `Naive UI`
- 路由继续使用 `vue-router`
- 状态和数据访问继续使用现有前台实现模式
- 样式优先复用 `apps/web-front/src/styles/main.css` 的结构和响应式方式

### 3. 不引入无必要的新 UI 体系

- 首批前台不引入与现有项目无关的第二套主组件库
- 不将后台管理页的 `Element Plus` 视觉直接搬到前台
- 若后续需要引入 `Tailwind CSS` 做布局增强，必须以“不破坏现有前台实现、可渐进接入”为前提，不作为首批开发的强制前置条件

### 4. 设计还原优先级

前台实现时，视觉还原优先级如下：

1. 信息结构和模块顺序
2. 交互入口和页面主链路
3. 响应式布局方式
4. 视觉层级、间距、留白、卡片关系
5. 图标、插图、示意图片等装饰性元素

如设计稿与已确认业务边界冲突，以项目文档为准；如桌面稿与移动稿表达不一致，以“功能不丢失、移动端可操作”原则处理。

## 移动端适配规则

### 1. 断点规则

首批适配按以下断点执行：

- 手机：`< 768px`
- 平板：`768px - 1023px`
- 桌面：`>= 1024px`

### 2. 导航规则

- 桌面端保持顶部主导航
- 手机端主导航改为折叠菜单、抽屉菜单或可展开导航
- 手机端不能继续使用整排横向挤压链接作为最终交付形态

### 3. 首页规则

- 搜索框在手机端应占据首屏主要位置
- 分类入口、资料中心、报价中心、公司库入口在手机端采用纵向堆叠或双列卡片
- 首页首批不出现热门产品与最新动态模块

### 4. 产品列表页规则

- 手机端不得直接照搬桌面宽表格
- 手机端产品列表使用卡片式列表
- 筛选区改为抽屉、底部弹层或独立筛选面板入口
- 每张卡片至少展示型号、名称、分类或标签、资料入口、报价摘要、关联公司、更新时间中的核心信息

### 5. 分类 / 品牌导航页规则

- 手机端以纵向区块展示分类入口和品牌入口
- 品牌索引可以分段折叠或按字母分组展示
- 不能要求用户在手机端进行大范围横向扫描

### 6. 商品详情页规则

- 手机端详情区块按纵向堆叠展示
- 规格参数、相关资料、报价记录、关联公司保持独立区块
- 右侧栏信息需要下沉到主内容之后，不保留桌面双栏结构
- 如信息较多，可使用 `Tabs`、分段按钮或折叠区块，但不能隐藏核心信息

### 7. 组件映射规则

移动端设计必须能映射到当前项目可实现的组件模式，例如：

- 搜索框：`NInput`
- 主要按钮：`NButton`
- 卡片容器：`NCard`
- 标签状态：`NTag`
- 栅格和区块：`NGrid`、`NGridItem` 或原生 CSS Grid
- 抽屉 / 弹层筛选：优先采用 Naive UI 可落地交互组件
- 说明信息：`NDescriptions`、列表卡片、分组信息块

不使用明显超出当前项目能力的复杂异形控件或实验性交互。

## 页面级验收规则

### 首页验收

- 与桌面稿和移动稿的模块顺序一致
- 搜索入口清晰可见
- 分类入口、资料中心入口、报价中心入口、公司库入口完整存在
- 不出现热门产品、最新动态

### 产品列表页验收

- 桌面端结构对齐 `product-list.png`
- 手机端结构对齐 `mobile-design-board-v1.png` 中的列表卡片表达
- 筛选功能可用，且手机端筛选入口易发现
- 列表信息中可直接感知资料、报价、关联公司信息

### 分类 / 品牌导航页验收

- 桌面端具备聚合导航能力
- 手机端可单手纵向浏览主要分类和品牌入口
- 分类、品牌结果跳转链路可用

### 商品详情页验收

- 桌面端结构对齐 `product-detail.png`
- 手机端结构对齐 `mobile-design-board-v1.png` 中的详情页表达
- 规格参数、相关资料、报价记录、关联公司四大区块完整
- 信息不是只有视觉占位，必须接入真实业务字段

## 交付验收规则

前台页面完成时，验收不只看截图，还要看是否满足以下条件：

- 页面结构与设计稿一致
- 移动端和桌面端都可操作
- 核心功能链路完整
- 不额外引入超出当前 MVP 范围的模块
- 使用项目现有组件库和可落地交互方式实现
- 数据展示基于真实业务字段，而不是纯静态占位

## 开发建议

前台首批开发建议顺序：

1. 公共壳层与导航
2. 首页
3. 产品列表页
4. 分类 / 品牌导航页
5. 商品详情页

如后续补新设计稿，应同步更新本文件和 `design/frontend-ui/README.md`，确保设计基线和验收口径保持一致。
