# 前台 UI 设计规范

本文件用于定义前台页面的通用视觉样式基线，优先服务首页、产品列表页、分类导航页、商品详情页等核心页面。

## 规范配图

- [ui-style-guide-board.png](./ui-style-guide-board.png)
  - 前台样式规范板，包含色彩、字体、间距、圆角、阴影、按钮、输入框、标签和状态样式示例。

## 适用原则

- 前台视觉以信息清晰、结构稳定、便于扩展为优先。
- 整体风格保持克制、专业、轻量，不追求过度装饰。
- 页面之间共享统一的颜色、字号、间距、圆角、边框和阴影规则。
- 设计规范优先定义通用样式，不在主规范内展开具体业务功能细节。
- 如页面设计稿与项目需求冲突，以已确认需求边界为准。

## 色彩规范

- 品牌主色：`#1664D9`
- 主交互色：`#1677FF`
- 深蓝强调：`#0B70D8`
- 主文字色：`#12213D`
- 常规文字色：`#52627D`
- 次级文字色：`#60708D`
- 辅助文字色：`#687895`
- 页面背景色：`#F4F7FB`
- 卡片背景色：`#FFFFFF`
- 弱化背景色：`#F7FAFF`
- 主边框色：`#E1E8F3`
- 分割线色：`#EDF2F8`
- 价格强调色：`#F05A1A`
- 状态色仅用于成功、警示、错误等语义提示，不替代品牌主色。

## 字体规范

- 字体族：`'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif`
- 页面主标题：`26px / 700`
- 区块标题：`20px / 700`
- 面板标题：`16px - 17px / 600`
- 导航文字：`16px / 600`
- 正文：`14px / 400 - 500`
- 辅助说明：`13px / 400`
- 关键数字或价格：`20px - 28px / 700 - 800`
- 同一页面内的字号层级应稳定，不随模块任意变化。

## 间距规范

- 基础间距建议采用：`4 / 8 / 12 / 16 / 24 / 32`
- 页面容器左右留白在桌面端保持统一。
- 模块与模块之间使用统一主间距，不因页面不同随意变化。
- 卡片、表格、筛选区、表单区内部使用一致的内边距节奏。
- 标签、按钮、图标与文字之间保持统一的微间距规则。

## 圆角与边框规范

- 卡片圆角：`8px`
- 大型面板或聚合容器圆角：`12px`
- 输入框、按钮圆角：`6px - 8px`
- 标签或胶囊类元素圆角：`999px`
- 常规边框：`1px solid #E1E8F3`
- 分割线：`1px solid #EDF2F8`
- 页面分层优先使用边框和浅底，不依赖厚重描边。

## 阴影规范

- 阴影仅用于浮层、悬停态和重点卡片。
- 默认卡片可无阴影或仅使用极轻阴影。
- Hover 阴影保持轻量，避免大面积悬浮感。
- 不允许在同一页面叠加多种强阴影风格。

## 常见组件样式

- 按钮：区分主按钮、次按钮、文字按钮，主按钮使用品牌蓝体系。
- 输入框：白底、浅边框、聚焦态转主色边框。
- 标签：用于分类、状态、属性补充信息，视觉应轻量。
- 卡片：白底、浅边框、统一圆角，作为主要信息容器。
- 表格：表头与内容区层级清晰，优先浅底和分割线表达结构。
- 导航：当前项使用蓝色高亮，未激活项保持深色或中性色。
- 分页、筛选、排序等控件样式应与输入框和按钮体系保持一致。

## 状态规范

- 组件需覆盖默认、悬停、激活、聚焦、禁用、加载六类状态。
- 禁用态应明确弱化，不可与可点击态过于接近。
- 错误、空态、加载态需具备统一反馈方式。
- 状态表达优先依靠颜色、描边和文案，不依赖复杂动画。

## 响应式规范

- 手机：`< 768px`
- 平板：`768px - 1023px`
- 桌面：`>= 1024px`
- 桌面端保持完整导航和高信息密度布局。
- 平板端优先压缩横向结构，保留核心内容。
- 手机端采用纵向堆叠、抽屉筛选、卡片化信息表达。
- 移动端不得直接沿用桌面宽表格作为最终交付形式。

## 使用要求

- 前台新增页面优先复用本规范，不单独发明新样式体系。
- 如需新增颜色、字号或组件样式，应先补充规范，再进入实现。
- 主规范保持简洁，具体页面示例可另行整理为补充文档。
