# 前台 UI 设计规范

本文件用于定义前台页面的通用视觉样式基线和首批页面结构约束，优先服务首页、产品列表页、分类导航页、系列详情页、资料中心页和报价中心页。  
当前阶段目标是完成文档级设计定稿，不进入像素级开发还原。

## 规范配图

- [ui-style-guide-board.png](./ui-style-guide-board.png)
  - 前台样式规范板，包含色彩、字体、间距、圆角、阴影、按钮、输入框、标签和状态样式示例。

## 适用原则

- 前台视觉以信息清晰、结构稳定、便于扩展为优先。
- 整体风格保持克制、专业、轻量，不追求过度装饰。
- 页面之间共享统一的颜色、字号、间距、圆角、边框和阴影规则。
- 设计规范不仅定义通用样式，也定义首批页面的模块顺序和信息承载边界。
- 如页面设计稿与项目需求冲突，以已确认需求边界为准。

## 色彩规范

- 品牌主色：`#1664D9`
- 主交互色：`#1677FF`
- 深蓝强调：`#0B70D8`
- 主文字色：`#12213D`
- 常规文字色：`#52627D`
- 次级文字色：`#60708D`
- 辅助文字色：`#687895`
- 页面背景色：`#F4F7FB`
- 卡片背景色：`#FFFFFF`
- 弱化背景色：`#F7FAFF`
- 主边框色：`#E1E8F3`
- 分割线色：`#EDF2F8`
- 价格强调色：`#F05A1A`
- 状态色仅用于成功、警示、错误等语义提示，不替代品牌主色。

## 字体规范

- 字体族：`'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif`
- 页面主标题：`26px / 700`
- 区块标题：`20px / 700`
- 面板标题：`16px - 17px / 600`
- 导航文字：`16px / 600`
- 正文：`14px / 400 - 500`
- 辅助说明：`13px / 400`
- 关键数字或价格：`20px - 28px / 700 - 800`
- 同一页面内的字号层级应稳定，不随模块任意变化。

## 间距规范

- 基础间距建议采用：`4 / 8 / 12 / 16 / 24 / 32`
- 页面容器左右留白在桌面端保持统一。
- 模块与模块之间使用统一主间距，不因页面不同随意变化。
- 卡片、表格、筛选区、表单区内部使用一致的内边距节奏。
- 标签、按钮、图标与文字之间保持统一的微间距规则。

## 圆角与边框规范

- 卡片圆角：`8px`
- 大型面板或聚合容器圆角：`12px`
- 输入框、按钮圆角：`6px - 8px`
- 标签或胶囊类元素圆角：`999px`
- 常规边框：`1px solid #E1E8F3`
- 分割线：`1px solid #EDF2F8`
- 页面分层优先使用边框和浅底，不依赖厚重描边。

## 阴影规范

- 阴影仅用于浮层、悬停态和重点卡片。
- 默认卡片可无阴影或仅使用极轻阴影。
- Hover 阴影保持轻量，避免大面积悬浮感。
- 不允许在同一页面叠加多种强阴影风格。

## 常见组件样式

- 按钮：区分主按钮、次按钮、文字按钮，主按钮使用品牌蓝体系。
- 输入框：白底、浅边框、聚焦态转主色边框。
- 标签：用于分类、状态、属性补充信息，视觉应轻量。
- 卡片：白底、浅边框、统一圆角，作为主要信息容器。
- 表格：表头与内容区层级清晰，优先浅底和分割线表达结构。
- 导航：当前项使用蓝色高亮，未激活项保持深色或中性色。
- 分页、筛选、排序等控件样式应与输入框和按钮体系保持一致。

## 状态规范

- 组件需覆盖默认、悬停、激活、聚焦、禁用、加载六类状态。
- 禁用态应明确弱化，不可与可点击态过于接近。
- 错误、空态、加载态需具备统一反馈方式。
- 状态表达优先依靠颜色、描边和文案，不依赖复杂动画。

## 响应式规范

- 手机：`< 768px`
- 平板：`768px - 1023px`
- 桌面：`>= 1024px`
- 桌面端保持完整导航和高信息密度布局。
- 平板端优先压缩横向结构，保留核心内容。
- 手机端采用纵向堆叠、抽屉筛选、卡片化信息表达。
- 移动端不得直接沿用桌面宽表格作为最终交付形式。

## 页面结构规范

### 首页

首页模块顺序建议：

1. 顶部搜索与快捷对象入口
2. 分类入口
3. 热门系列
4. 资料中心入口
5. 最新报价批次
6. 最新动态

要求：

- 首页必须强调真实产品入口，不只做通用快捷导航。
- 最新报价模块展示对象是“报价批次”，不是单个价格数字。

### 产品列表页

页面结构建议：

1. 顶部筛选区
2. 排序与结果统计
3. 系列卡片或系列列表
4. 侧边或抽屉式高级筛选

首批筛选项至少包括：

- 分类
- 品牌
- 尺寸
- 芯片
- RAM
- 存储
- 亮度
- POE
- 报价状态

卡片或列表项至少展示：

- 系列名称
- 代表图
- 变体数量
- 尺寸范围或主要尺寸
- 关键芯片/内存摘要
- 是否有有效报价
- 资料数量

### 系列详情页

详情页主对象是系列，不是单型号。  
页面模块顺序建议：

1. 系列头部信息
2. 变体切换器
3. 核心参数对比
4. 完整规格分组
5. 阶梯报价
6. 选配说明
7. 图片/图纸
8. 相关资料
9. 最近动态

要求：

- 变体切换器必须足够明显，支持在 6 个型号之间快速切换。
- 核心参数对比只展示高频字段，如尺寸、芯片、RAM、存储、系统、亮度、POE。
- 完整规格必须保留原始分组语义，如 `System`、`Display`、`Touch`、`Network`。
- 报价区必须能展示 `10 / 100 / 500 / 1000` 等阶梯价。
- 选配说明必须能表达：
  - 支架类选配
  - 固件包说明
  - 镀锌板减价等差价项
- 图片区和图纸区可以共用视觉语言，但信息语义必须区分。

### 报价中心页

页面结构建议：

1. 顶部筛选区
2. 报价批次列表
3. 批次展开或详情查看入口

首批筛选项至少包括：

- 公司
- 系列
- 变体
- 发布时间
- 状态

列表列建议：

- 报价批次标题
- 公司
- 覆盖变体数
- 币种
- 发布时间
- 生效状态
- 来源文件

批次详情至少展示：

- 批次头信息
- 变体报价行
- 各报价行阶梯价
- 选配/减价说明
- 原始工作簿入口

### 资料中心页

资料中心必须按类型分组展示，而不是统一文件流。  
首批类型至少包括：

- 原始报价工作簿
- 规格资料
- 产品图片
- 尺寸图/结构图
- 认证或技术资料

列表或卡片至少展示：

- 标题
- 关联系列/变体
- 文档类型
- 来源公司
- 更新时间

### 公司库

公司详情页应能承接：

- 公司基础信息
- 关联系列
- 最近报价批次
- 平台相关动态

## 后台结构参考

虽然本文件主要服务前台，但后台页面结构也需要与前台对象模型一致。

### 后台系列管理

- 主列表对象是系列。
- 系列详情页内维护：
  - 系列基础信息
  - 变体表格
  - 规格项入口
  - 关联媒体与资料

### 后台报价批次管理

- 主列表对象是报价批次。
- 批次详情页内维护：
  - 批次头信息
  - 报价行
  - 阶梯价
  - 选配项
  - 原始 workbook 关联

### 后台文档管理

- 必须支持区分：
  - 原始 workbook
  - 图片
  - 图纸
  - 规格文件
  - 认证资料

## 文档级交付标准

当前阶段页面设计只要求做到低保真或中保真决策完成：

- 模块顺序明确
- 关键字段展示明确
- 表格列设计明确
- 筛选项明确
- 编辑表单结构明确

当前阶段不要求：

- 像素级定稿
- 组件代码实现
- 交互动画打磨

## 使用要求

- 前台新增页面优先复用本规范，不单独发明新样式体系。
- 如需新增颜色、字号或组件样式，应先补充规范，再进入实现。
- 如需调整页面模块顺序或对象边界，应先同步更新 `PRD`、`ARCHITECTURE`、`DATA_MODEL`、`ROUTES` 后再进入开发。
