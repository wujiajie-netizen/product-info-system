# 前台 UI 设计稿

本目录存放前台页面的 UI 设计参考图和页面结构说明，供 Codex 和开发人员在实现前台页面时读取。  
当前阶段以“文档级页面设计”定稿为目标，不进入像素级实现还原。

## 使用原则

- 涉及前台页面开发时，优先按本目录中的设计图和结构说明实现页面模块、信息层级和交互顺序。
- 如设计图与项目文档冲突，以 `docs/PRD.md`、`docs/ARCHITECTURE.md`、`docs/DATA_MODEL.md`、`docs/ROUTES.md` 中已确认的需求边界为准。
- 当前设计图主要用于前台主链路，不代表所有个人协作能力都已进入 MVP。
- 前台开发与验收时，同时参考 [UI_SPEC.md](./UI_SPEC.md) 中的布局规则、组件约束、设计令牌、页面模块和验收规则。

## 问答中心前台页说明

- 页面入口名称：`问答中心`。
- 页面说明文案：为您整理系统使用过程中的常见问题，帮助您快速找到解决方案。
- 核心结构：
  - 搜索框：支持按问题关键词搜索。
  - 分类标签：全部、功能使用、账号权限、产品信息、数据维护、常见异常。
  - 左侧主内容：以 Q/A 折叠面板展示问题和答案，默认展开首条或高频问题。
  - 右侧辅助栏：展示热门问题，并提供“仍未解决？联系管理员”入口。
- 视觉风格：仅参考资讯模块的前台样式统一性，不采用资讯时间线布局。

## 页面结构结论

- 前台商品详情主对象是系列，不再设计为“单型号单页”。
- 型号以变体切换器呈现在系列详情页内。
- 报价中心主对象是报价批次和报价行，不再展示为单一 `unit_price` 列表。
- 资料中心必须区分：
  - 原始报价工作簿
  - 规格资料
  - 产品图片
  - 尺寸图/结构图
- 当前阶段只要求低保真或中保真决策完成：
  - 模块顺序
  - 关键字段展示
  - 表格列设计
  - 筛选项
  - 编辑表单结构

## 备注

- 这些图片为静态参考图，不是可运行页面。
- 前台不再单独设计聚合搜索结果页；首页搜索按对象跳转到对应列表页承接结果。
- 前台顶部导航不单列品牌入口；品牌导航归入分类页。
- 如后续设计稿继续增加，保持“按页面语义命名”的方式，不再使用 `image1.png` 这类无语义文件名。
- 后台页面虽然不在本目录保存视觉稿，但其页面结构和录入模块也应遵循 `UI_SPEC.md` 中的结构说明。
